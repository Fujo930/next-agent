# F — 投机工具执行 (SPECULATIVE) — 已废弃

## 结论：不可行

预执行工具调用在实践中不成立。

## 原因

1. **本地 I/O 不是瓶颈** — 读文件 1-3ms，但 DeepSeek API 延迟 500ms-2s。预读文件省不了时间
2. **预测准确率低** — DeepSeek 下一步"可能"读什么文件无法可靠预测
3. **上下文浪费** — 提前读的数据放哪？放 context 浪费空间；缓存等 LLM 问就用普通缓存
4. **副作用** — git 命令、shell 执行有副作用，不能投机执行

## 被取代的方案

用**工具结果缓存**代替：
- 同一轮内同工具+同参数 → 返回缓存结果
- 不执行推测性操作
